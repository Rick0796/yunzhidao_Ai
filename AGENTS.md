# 项目长期规则

## 1. 项目定位
这是我们团队内部使用的短视频内容增长工作流系统。
它不是公开 SaaS，不是通用 AI 平台，也不是全自动发布工具。

## 2. 我们的业务
我们服务实体老板、中小企业、创业者，帮助他们做 AI 获客增长、内容增长、客户增长、企业增长。

## 3. 我们当前的真实工作方式
我们已经有：
1. 数据来源：禅妈妈、飞瓜等抖音数据网站，用来找全平台热点视频和同行业爆款视频
2. 文案改写网站：`https://fange-technology.vercel.app/`
3. 数字人网站：可把文案生成数字人视频
4. 人工团队：负责文案终选、视频审核、视频剪辑、发布、复盘

## 4. 当前目标
先把我们自己的内部短视频生产流程 agent 化、流程化、可追踪化。
第一版先做半自动生产线，不追求一步到位全自动。

## 5. 当前核心流程
禅妈妈/飞瓜发现热点或同行爆款视频
-> 判断视频基础数据是否值得跟进
-> 提取原视频文案
-> 送入凡哥科技网站的“短视频文案分析”
-> 输入个人/业务背景介绍与可选具体需求
-> 生成多条新文案
-> 选择文案送入数字人网站生成视频
-> 下载视频到本地
-> 人工审核、剪辑、发布
-> 记录结果并复盘

## 6. 当前必须保留人工的环节
- 选题最终确认
- 最终文案确认
- 视频审核
- 视频剪辑
- 发布
- 复盘

## 7. 开发原则
- 先做最小可用版本
- 先做真实流程的结构和状态，不做过度抽象
- 能接真实网页流程的地方优先接真实流程
- 没有稳定接口的地方再用 mock 兜底
- 模块化、可替换、可扩展
- 本地可运行
- 有日志
- 有失败重试
- 有人工确认节点

## 8. 当前不做
- 不做全自动发布
- 不做大而全 SaaS 架构
- 不做复杂权限系统
- 不做计费系统
- 不做多租户
- 不做重型前端美化
- 不做逆向接口和反爬对抗

## 9. 内容方向
我们当前主要处理三类短视频内容：
1. 平台热点事件视频
2. 同行业爆款视频
3. AI 获客、创业者成长、企业增长、内容获客、实体老板增长相关视频

## 10. 文案生成要求
最终输出主要用于数字人短视频，所以文案需要：
- 短句
- 口语化
- 有钩子
- 有价值
- 有结果感
- 适合直接送入数字人生成

## 11. 技术倾向
优先使用：
- Python
- FastAPI
- SQLAlchemy
- SQLite 或 PostgreSQL
- 简单内部后台
- 模块化 workflow / agent 结构
- 可插拔网页适配层

## 12. 自动化边界
允许做：
- 对禅妈妈/飞瓜的受控半自动采集或录入
- 对凡哥科技网站的受控页面操作
- 对数字人网站的受控页面操作
- 生成后下载视频到本地

不做：
- 绕过登录
- 破解接口
- 大规模爬虫系统
- 无人工确认的全流程自动发布

## 13. 成功标准
成功的定义是：
团队能稳定跑通这条链路：
发现视频 -> 判断数据 -> 提取文案 -> 改写生成 -> 数字人视频生成 -> 下载本地 -> 审核 -> 剪辑 -> 发布 -> 复盘
