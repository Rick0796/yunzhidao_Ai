# 开发规则

## 开发原则
先做最小可用版本，先把真实业务链路跑通。

## 第一版必须包含
- 清晰的任务状态机
- 数据库表结构
- 模块化 workflow / agent
- 内部后台
- 真实网页适配层或可替换适配层
- 必要的 mock 兜底能力
- 日志
- 失败重试
- 本地运行能力
- 本地文件下载与保存能力

## 第一版不要做
- 自动发布
- 通用型复杂浏览器自动化平台
- 复杂权限系统
- 计费系统
- 多租户 SaaS
- 过度设计的基础设施
- 很重的前端美化
- 逆向接口
- 反爬对抗

## 当前工作流目标
支持这条链路：
禅妈妈/飞瓜发现视频
-> 判断基础数据
-> 提取原视频文案
-> 发送到凡哥科技网站改写
-> 选中生成结果
-> 发送数字人网站生成视频
-> 下载本地
-> 人工审核剪辑
-> 发布记录
-> 数据复盘

## 接口原则
- 优先接已有真实网页流程
- 没有稳定接口的地方再用 mock
- 所有外部能力都要做成可替换适配层
- 不假设外部网站有官方 API

## 代码原则
- 清晰优先
- 模块优先
- 状态明确
- 尽量配置化
- 易于扩展
- 先围绕真实业务流程建模

## UI 原则
内部后台只要简单可用。
功能优先，不追求美观。

## 错误处理
任何外部步骤都要支持：
- 状态记录
- 日志
- 错误信息
- 重试
- 人工接管

## 重要限制
- 不要假设禅妈妈、飞瓜、凡哥科技、数字人网站有完整开放 API
- 请做成可替换适配层
- 允许受控页面操作，但不要做重型通用自动化框架
- 允许下载生成后的视频到本地

## 数据要求
每一条任务都要能从“发现视频”追踪到“本地视频结果”与“发布结果”。

## 权限边界
允许：
- 正常网页访问
- 正常页面填写与点击
- 下载文件到本地工作目录

不允许默认假设：
- 绕过登录
- 非法抓取
- 平台对抗
